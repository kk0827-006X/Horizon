## [Guide to Free *.city.state.us Locality Domains](https://fredchan.org/blog/locality-domains-guide/) ⭐️ 7.0/10

A comprehensive guide published in 2025 explains how to obtain a free locality domain under the .us TLD, specifically in the format city.state.us. The guide details the process of finding delegated locality managers and registering domains at no cost. This guide is valuable for individuals and small organizations seeking memorable, localized domain names without annual fees. It highlights the bureaucratic hurdles and technical obstacles that have historically made such domains difficult to obtain, which the community discussion confirms. Locality domains require registrants to contract with a Delegated Manager, and some managers are defunct, requiring persistence or alternative registration. The guide notes that WHOIS privacy is forbidden for .us domains, which poses a privacy risk for personal use.

hackernews · speckx · May 13, 14:45 · [Discussion](https://news.ycombinator.com/item?id=48122635)

**Background**: The .us TLD originally allowed only fourth-level registrations in the form organization-name.locality.state.us, based on political geography as per RFC 1480. Second-level registrations were introduced in 2002, but locality domains persist as a special structure managed by delegated managers for each city/state combination.

## [LLM 0.32a2 Adds Support for OpenAI Reasoning Endpoint](https://simonwillison.net/2026/May/12/llm/#atom-everything) ⭐️ 6.0/10

LLM 0.32a2 now routes reasoning-capable OpenAI models through the /v1/responses endpoint instead of /v1/chat/completions, enabling interleaved reasoning across tool calls. Users can view summarized reasoning tokens in a different color and hide them with -R flag. This update improves transparency for users of reasoning models like GPT-5, making it easier to understand model decision processes. It also prepares LLM for future OpenAI API changes as the /v1/responses endpoint becomes the standard for stateful interactions. This is an alpha release (0.32a2), so stability is not guaranteed. The /v1/responses endpoint supports stateful interactions and built-in tools like file search and web search.

rss · Simon Willison · May 12, 17:45

**Background**: LLM is a command-line tool by Simon Willison for running prompts against various language models. The /v1/responses endpoint is OpenAI's newer API for creating stateful interactions with models, replacing the older /v1/chat/completions endpoint for reasoning models. Reasoning tokens are internal steps the model uses to arrive at an answer, which some models can expose for greater transparency.
