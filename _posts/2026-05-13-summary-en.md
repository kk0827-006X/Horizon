## [Needle: a 26M parameter tool calling model for on-device AI](https://github.com/cactus-compute/needle) ⭐️ 8.0/10

Needle is an open-source 26M parameter model that achieves fast tool calling on consumer devices using only attention and gating, without MLPs. It outperforms larger models like FunctionGemma-270M in single-shot tool calling accuracy. This demonstrates that tool calling can be framed as a retrieval task rather than reasoning, enabling much smaller models suitable for on-device agents. It makes agentic experiences feasible on budget phones, watches, and other edge devices. The model was pretrained on 200B tokens on 16 TPU v6e for 27 hours, then post-trained on 2B synthetic tokens of function-calling data generated by Gemini across 15 tool categories. It runs at 6000 tok/s prefill and 1200 tok/s decode on consumer hardware.

hackernews · HenryNdubuaku · May 12, 18:03 · [Discussion](https://news.ycombinator.com/item?id=48111896)

**Background**: Tool calling is the ability of an AI model to invoke external functions or APIs to perform actions beyond text generation. Traditional large language models use massive feed-forward networks (FFNs) to store factual knowledge, but Needle shows that for tool calling, the model only needs to match input to a tool template and extract arguments, which can be done efficiently with cross-attention and gating.

## [Using LLM in shebang lines for script execution](https://simonwillison.net/2026/May/11/llm-shebang/#atom-everything) ⭐️ 6.0/10

Simon Willison demonstrated a technique to use the LLM command-line tool in the shebang line of scripts, enabling scripts to generate outputs via LLM fragments and tool calls. This approach combines AI generation with traditional Unix scripting, allowing developers to create executable English text files that can perform complex tasks via LLM, potentially streamlining prototyping and automation. The simplest example uses `#!/usr/bin/env -S llm -f` followed by a prompt, while more advanced versions can include tool calls with `-T` and even define custom Python functions in YAML templates.

rss · Simon Willison · May 11, 18:48

**Background**: A shebang line (`#!`) at the top of a script tells Unix systems which interpreter to execute the file with. The `llm` tool is a command-line utility for interacting with large language models, supporting fragments (reusable text snippets) and tool calls (function-like extensions). By placing `llm` in the shebang, any text file can become an executable that sends its content as a prompt to an LLM.
